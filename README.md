# OntologyGenerGraph

OntologyGenerGraph — исследовательская система автоматической генерации графов знаний из текстовых данных с использованием больших языковых моделей (LLM) и онтологических ограничений.

Проект разработан в рамках исследования влияния онтологий на качество построения графов знаний. Система позволяет сравнивать различные стратегии извлечения знаний из текста и оценивать вклад онтологии в повышение качества получаемого графа.

## Цель проекта

Основной целью проекта является исследование методов ontology-guided knowledge graph generation и оценка влияния онтологических ограничений на качество автоматически генерируемых графов знаний.

В рамках проекта реализовано сравнение нескольких подходов:

* генерация графа знаний без использования онтологии;
* генерация графа знаний с использованием списка допустимых отношений;
* генерация графа знаний с использованием ограничений domain/range;
* генерация графа знаний с последующей онтологической валидацией.

## Используемый набор данных

Для проведения экспериментов используется набор данных **Text2KGBench (DBpedia-WebNLG)**.

Набор данных содержит:

* текстовые описания объектов предметной области;
* онтологии предметных областей;
* эталонные графы знаний в виде триплетов;
* тестовые и обучающие выборки.

В текущей реализации используется подмножество **University Ontology**.

## Архитектура системы

Система включает следующие основные компоненты:

* модуль загрузки данных;
* модуль взаимодействия с языковой моделью;
* модуль извлечения триплетов;
* модуль онтологической валидации;
* модуль расчёта метрик качества;
* модуль проведения экспериментов;
* модуль сохранения результатов.

В процессе работы текстовое описание передаётся языковой модели, которая извлекает триплеты графа знаний. Затем результаты могут быть проверены на соответствие онтологии, после чего рассчитываются метрики качества и сохраняются результаты эксперимента.

## Методы оценки качества

Для оценки качества генерации графов знаний используются следующие метрики:

### Strict Metrics

* Precision
* Recall
* F1-score

Данные метрики требуют полного совпадения триплета с эталонным графом знаний.

### Fact Metrics

* Fact Precision
* Fact Recall
* Fact F1

Метрики оценивают качество извлечения фактов независимо от точного названия отношения.

### Ontology Metrics

* Ontology Validity

Метрика показывает долю отношений, соответствующих онтологии предметной области.

## Настройка окружения

### Создание файла `.env`

В корневой директории проекта необходимо создать файл `.env`:

```env
CLOUDRU_API_KEY=<YOUR_API_KEY>
CLOUDRU_BASE_URL=https://foundation-models.api.cloud.ru/v1
```

## Запуск через Docker

### Сборка Docker-образа

```bash
docker build -t ontology-kg .
```

### Запуск эксперимента

```bash
docker run --env-file .env -v ${PWD}/outputs:/app/outputs ontology-kg
```

После завершения работы результаты будут сохранены в каталоге `outputs`.

## Выходные файлы

После выполнения эксперимента формируются следующие файлы:

### results_full.json

Содержит подробные результаты генерации для каждого примера набора данных.

### summary.json

Содержит усреднённые значения метрик качества по всем экспериментам.

### metrics.csv

Содержит итоговую таблицу метрик для всех исследуемых методов.
